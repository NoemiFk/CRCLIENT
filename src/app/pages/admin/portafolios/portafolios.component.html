Paso 2. Administración Portafolios

  1. Carga los archivos de los portafolios a gestionar, presionando el botón.

  2. El archivo no debe contener espacios vacíos.
  3. El archivo debe estar en formato csv.
  1. Selecciona los campos para la segmentación, la comunicación y el portal del cliente. También puedes indicar los campos que corresponden al aval.
  2. Este paso es necesario que indiques al Collection Robot que campos quieres usar en cada etapa del proceso de gestión.
  3. Campos que se usan p.e. para la segmentación son: días de morosidad, monto cartera vencida o riesgo, estado, producto etc.
  4. Campos que se usan p.e para la comunicación son: Nombre y dirección del cliente, identificador del crédito, monto atraso, información del producto.
  5. Campos que se usan p.e para el portal del cliente son: Nombre, producto, identificador crédito, detalle cartera vencida, producto, identificador crédito, detalle cartera vencida.
  6. En caso de contar con avales se debe indicar los campos del aval como nombre, dirección, teléfono
  1. Carga en Collection Robot información valiosa para optimizar la gestión y recursos. Todos archivos siempre deben estar en formato csv.

Promesas de Pago: Sube las promesas de pago que obtienes para que Collection Robot considere y suspenda el envío de comunicación hasta obtener información sobre el cumplimiento del pago. El archivo a cargar debe contener los campos de Identificador cliente, Fecha de Promesa y Monto de promesa.

Pagos: Actualiza los pagos que obtienes de los clientes. El Collection Robot considera los pagos para el envió de información de cartera vencida al cliente y para medir el cumplimiento de las promesas de pago

Formato: Identificador cliente, Fecha de Pago, Monto de pago

Resultados de Gestión: Formato: Identificador cliente, Fecha gestión, Canal gestión, Resultado (éxito, sin éxito)
